Reflexion: 
* Conviene trabahar con ramas al corregir errores, realizar pruebas q no afecten a la rama principal, o para trabajar entre varias personas sin afectar la rama principal
* git reset Deshace commits y mueve el historial hacia atras para conservar o eliminar cambios realizados
* git revert crea un nuevo commit que anula el anterior a este.
* git checkout permite cambiar entre ramas creadas, recuperar archivos o ver versiones anteriores de un protecto
* Los mensajes cortos de commits (Excepto por el q cree, 6ffdbcc), son utiles para explicar que cambios se realizaron y que el historial sea mas intuitivo y entendible.
* Git es de gran ayuda ya que cada integrante de un proyecto puede trabajar desde su computadora en diferentes sectores del proyecto y en lo que mejor se le de, ya sea en programacion con html, Css, C++, entre otros lenguajes, de ahi pueden subir todo a las plataformas y comparar versiones, arreglar conflictos, entre otros, sin perder el historial de cada cambio realizado en el mismo.
